## Mandatory map: `postN` ↔ `maturity_build`

Per ADR-0073 (ratified 2026-06-27): without this table, publication and maturity counters diverge without audit trail.

| PyPI / `[project] version` | `maturity_build` (octet) | Notes |
| --- | --- | --- |
| **`1.7.4`** | **`.201`** | GA on PyPI (immutable); release PR **#1024** |
| **`1.7.4.post1`** | **`.202`** | SQL drivers → optional extras; lean core `pip install` ([#1047](https://github.com/FabioLeitao/data-boar/issues/1047)) |

**Divergence example (policy):** If fixes advance the octet without a PyPI upload, the publish counter stays on the last **`postN`** until republication. After **`1.7.4.post1`** was published at **`.202`**, fixes **`.203`–`.208`** accumulated on `main`; the next upload is **`1.7.4.post2`** ↔ **`.208`** — see [`1.7.4.post2.md`](1.7.4.post2.md).

## Highlights

- **Lean core install:** C-extension SQL drivers removed from default dependencies; install via extras (`postgres`, `mysql`, `mariadb`, `mssql`, `oracle`, `sql-all`).
- **Lazy-import guards:** Clear `ImportError` with `pip install 'data-boar[<extra>]'` when a connector needs a missing driver.
- **MariaDB dialect:** `driver: mariadb` maps to SQLAlchemy **`mariadb+mariadbconnector`** (MariaDB Connector/C extra).

pip install 'data-boar==1.7.4.post1'
