# ADR 0056 - Cryptographic ADR inventory (inv-adr.ps1 + SSH ed25519 attestation)

**Status:** Accepted
**Date:** 2026-05-20

## Context

Git history proves *when* files changed but does not give third parties a portable, file-level integrity manifest without full repository access. The project maintains many Architecture Decision Records under `docs/adr/`. Contributors and agents need a deterministic inventory of each `ADR-*.md` with per-file SHA-256 and a self-attested aggregate hash.

`scripts/inv-adr.ps1` (issue **#575**) generates `docs/adr/INVENTORY.txt` with per-ADR hashes and an `InventoryHash` over data rows. The *policy* for attestation, signing, and PR hygiene was not yet an ADR.

## Rejected alternatives

| Option | Why not |
| ------ | ------- |
| **Git-only provenance** | `git log` is not verifiable by third parties without clone access and does not hash individual ADR bodies in one artifact. |
| **GPG detached signatures** | Valid, but the operator already uses SSH ed25519 keys registered with GitHub; avoids parallel web-of-trust tooling. |
| **Inventory without hashes** | A filename list would be cosmetic with no forensic value. |

## Decision

1. Treat `docs/adr/INVENTORY.txt` as the **canonical integrity manifest** for all `ADR-*.md` files, regenerated by `.\scripts\inv-adr.ps1` after any ADR add, edit, or rename.
2. Use **SSH ed25519** detached signatures (`ssh-keygen -Y sign`) on `INVENTORY.txt`, stored as `INVENTORY.txt.sig`, for operator attestation when publishing evidence (homelab or release hygiene).
3. Keep **`InventoryHash`** as a self-attested SHA-256 over inventory data rows (excluding comment lines and the hash line itself), enforced by `tests/test_scripts.py`.
4. **`scripts/new-adr.ps1`** runs `inv-adr.ps1` after scaffolding; it also prints an explicit reminder to re-run inventory when ADRs are edited outside that script.
5. **CI** asserts inventory completeness and hash consistency via pytest; it does **not** hold the operator private key for signing.

### Guardrails

- **NEVER** delete `INVENTORY.txt` or `INVENTORY.txt.sig` from the tree without a replacement ADR and a documented re-sign ritual.
- **Every** PR that changes any `docs/adr/ADR-*.md` must update `INVENTORY.txt` in the **same PR** (and `INVENTORY.txt.sig` when the operator re-signs).
- **Key rotation** on ed25519 invalidates prior `.sig` files; re-sign the current `INVENTORY.txt`, note rotation in the inventory header comment block, and reference this ADR.
- Assistants: if unsure whether inventory is current, run `.\scripts\inv-adr.ps1` before commit.

## Consequences

- **Positive:** Auditors can verify ADR corpus integrity from two artifacts; agents have a machine-checkable sync test.
- **Negative:** Extra step on every ADR touch; signature is operator-local (not CI-mandatory).
- **Watch:** Legacy ADRs without inventory rows are caught by pytest; do not bypass with `--no-verify` unless intentional.
